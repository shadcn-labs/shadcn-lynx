# Contributing to shadcn-lynx

We appreciate your interest in contributing. This guide will help you get started with the contribution process.

## Ways to contribute

We welcome all kinds of contributions:

- **Feedback** - have suggestions or ideas? Open a discussion to talk about it. This project is a port of [shadcn/ui](https://github.com/shadcn-ui/ui) for Lynx.
- **Bug reports** - please open an issue to report a bug.
- **Docs fixes** - spot a typo or broken link? Open a PR.
- **Code improvements** - For anything more involved than a quick fix, start a discussion first so we can align before you dive in.

## How to contribute

We try to keep the process simple to avoid wasted effort:

### 1. Reporting bugs

Open an issue using the bug report template. Make sure to provide a minimal reproduction of the issue.

### 2. Requesting missing features from `shadcn/ui`

Before requesting a feature please check if it is already in [shadcn/ui](https://github.com/shadcn-ui/ui). If it is, please open a feature request issue.

### 3. Opening PRs

Thanks for contributing to `shadcn-lynx`!

We ask that you follow these guidelines to help make it easier for us to review your PR:

- Your PR should fix a _single issue_ linked in the PR description (e.g. `Fixes #123`)
- Your PR should pass all CI checks
- Format & lint your code with `pnpm format` and `pnpm check`
